- O objetivo do projeto é analisar o churn de uma empresa de telecomunicações. Extrair, tratar, modelar e organizar a base. Calcular métricas e entregar um produto que auxilie na tomada de decisão.
- Para isso, pedi para uma IA agir como o Stakeholder (gerou o arquivo Stakeholder_anatomia_churn), explicando os seus objetivos, os números que deseja enxergar e as decisões que planeja tomar com base nos dados que serão entregues.
- Dessa forma, é possível simular um cenário de um projeto real de dados no mercado de trabalho.

- Primeiro passo foi desenhar a modelagem para tentar compreender os relacionamentos e quais tabelas serão úteis para alcançar as respostas que eu busco.
- Após estudar as bases, montei todo o esquema e modelo do banco, construindo relacionamentos e ajustando campos.
- Vou começar a explorar as tabelas e como entender quais dados preciso usar para fazer a primeira análise, a mais simples, churn mensal.
- Após criação do documento de churn mensal, usando o script calculo_churn_mensal.py, conseguimos ter uma boa visão do comportamento do churn por meses e anos.
- Agora vou seguir para segmentar o churn por planos, e depois segmentar por meses e planos juntos.
- Antes de seguir para os planos, tive que corrigir os nomes das colunas na segmentação padrão por meses.
- Pronto, script calculo_churn_plano.py funcionando.
- Novamente algumas correções nas colunas foram necessárias, tanto em churn_mensal.csv quanto em churn_mensal_por_plano.csv.
- Com as correções feitas, fiz o script calculo_churn_uf.py, que segmenta a taxa de churn por estado brasileiro.
- Segui para o último ponto a ser resolvido para concluir a primeira etapa (DECISÃO 1 no documento do STAKEHOLDER). Esse ponto é criar a análise de churn por tempo de casa.
- Script de calculo_churn_tempo_de_casa.py concluído. Gera o arquivo churn_mensal_por_tempo_de_casa.csv, auxiliando na análise do tempo de vida do cliente na empresa.

- Iniciando a decisão 2, vou pensar em como estruturar as métricas necessárias para entender os motivos por trás do churn. (atraso no pagamento, muitos suportes etc)
- Determinei uma janela de 90 dias para analisar os acontecimentos pré-cancelamentos.
- Script de base_analise_motivos.py concluído. Seu objetivo é gerar o csv esqueleto (base_contratos_stauts_churn.csv) para calcular os indicadores e entender o que mais parece causar churn.
- Agora vamos começar a analisar e entender os possíveis geradores de churn.
- A primeira relação que iremos analisar é de Inadimplência / Atraso. Quanto mais inadimplência mais chance de cancelamento? Quanto mais faturas atrasadas mais chance de cancelamento? Responderemos essas perguntas.
- Subi os CSVs em um modelo no Power BI (salvo na pasta bi/). Começarei os testes e a exploração dos dados com gráficos.
